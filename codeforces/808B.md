# Average Sleep Time

> [Problem on Codeforces](https://codeforces.com/problemset/problem/808/B)

## Idea

The key insight is that the sum of any $k$ consecutive elements in the array can be efficiently computed using a sliding window approach. Specifically, if we maintain the sum of the first $k$ elements, we can compute the sum of the next $k$ elements by subtracting the first element of the current window and adding the next element in the array. This avoids recomputing the sum from scratch for each window, reducing the time complexity significantly.

## Approach

1. Compute the sum of the first $k$ elements of the array and store it in a variable, say `cur`. This represents the sum of the first sliding window.

2. Iterate through the array starting from the $(k+1)$-th element. For each step, update the current sum `cur` by subtracting the element that is sliding out of the window (i.e., $a[i-k]$) and adding the element that is sliding into the window (i.e., $a[i]$). Accumulate this updated sum into a variable `total` to keep track of the sum of all sliding windows.

3. After processing all sliding windows, compute the average by dividing `total` by the number of windows, which is $n-k+1$.

4. Output the result with a precision of at least six decimal places to meet the problem's accuracy requirements.

## Complexity

- **Time:** $O(n)$ — computing the initial sum and iterating through the array with the sliding window both take linear time.
- **Space:** $O(1)$ — only a constant amount of extra space is required for variables like `cur` and `total`.

## Notes

- Using a sliding window approach ensures that we avoid recalculating sums redundantly, which would otherwise lead to an $O(n \cdot k)$ time complexity.
- The use of `fixed` and `setprecision(10)` ensures that the output meets the required precision constraints.
