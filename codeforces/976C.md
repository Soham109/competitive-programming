# Nested Segments

> [Problem on Codeforces](https://codeforces.com/problemset/problem/976/C)

## Idea

The key insight is that if one segment $[l_i, r_i]$ is nested within another segment $[l_j, r_j]$, then $l_i \geq l_j$ and $r_i \leq r_j$. Sorting the segments by their right endpoint $r$ in descending order ensures that for any given segment, all subsequent segments in the sorted order have $r \leq r_i$. This allows us to efficiently check if $l_i \geq l_j$ for a valid nesting condition, as only the left endpoints $l$ need to be compared sequentially.

## Approach

1. Represent each segment as a triplet $(r_i, l_i, i)$, where $i$ is the original index of the segment. This ensures that we can sort the segments by their right endpoint $r_i$ in descending order, breaking ties by sorting $l_i$ in ascending order.

2. Sort the segments as described above. Sorting by $r_i$ in descending order ensures that for any segment, all subsequent segments in the sorted list have smaller or equal $r$ values. Sorting by $l_i$ in ascending order for tied $r_i$ values ensures that the segment with the smallest $l$ is considered first.

3. Maintain a variable `prevlo` to track the smallest $l$ encountered so far and its corresponding index `prevloidx`. Iterate through the sorted list, and for each segment $[l_i, r_i]$, check if $l_i \geq$ `prevlo`. If true, $[l_i, r_i]$ is nested within the segment represented by `prevlo`.

4. If a valid pair is found, output the indices of the nested segment and the containing segment. If no such pair exists after processing all segments, output $-1, -1$.

## Complexity

- **Time:** $O(n \log n)$ — dominated by the sorting step.
- **Space:** $O(n)$ — for storing the segments and their indices.

## Notes

- Sorting by $r_i$ in descending order ensures that the outermost segments are considered first, simplifying the nesting check. This is the core of the greedy approach. 
- The use of `array<int,3>` in the implementation allows us to track the original indices while sorting by $r_i$ and $l_i$.
