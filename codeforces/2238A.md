# Another Puzzle from Papyrus

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2238/A)

## Idea

The key observation is that any valid transformation from $a$ to $b$ must satisfy $a_i \geq b_i$ for all $i$. This is because the only valid operation on individual elements of $a$ is subtraction, which can only decrease their values. If this condition is not met for any $i$, the transformation is impossible. Additionally, reordering $a$ incurs a fixed cost $c$, so we must decide whether reordering reduces the total cost of the transformations needed.

## Approach

1. **Check feasibility without reordering:**  
   Compare each $a_i$ with $b_i$. If $a_i < b_i$ for any $i$, the transformation is impossible without reordering. Otherwise, calculate the total cost of transforming $a$ to $b$ without reordering as $total1 = \sum_{i=1}^n (a_i - b_i)$.

2. **Check feasibility with reordering:**  
   Sort both arrays $a$ and $b$. After sorting, compare the elements pairwise. If any $a_i < b_i$ after sorting, the transformation is impossible even with reordering. Otherwise, calculate the total cost of transforming the sorted $a$ to $b$ as $total2 = c + \sum_{i=1}^n (a_i - b_i)$, where $c$ is the cost of reordering.

3. **Determine the minimum cost:**  
   If both transformations (with and without reordering) are possible, the answer is $\min(total1, total2)$. If only one transformation is possible, the answer is the corresponding cost. If neither transformation is possible, the answer is $-1$.

## Complexity

- **Time:** $O(t \cdot n \log n)$, where $t$ is the number of test cases and $n$ is the length of the arrays. Sorting dominates the complexity.
- **Space:** $O(n)$ per test case for storing the arrays.

## Notes

- Sorting the arrays ensures that the largest elements in $a$ are matched with the largest elements in $b$, minimizing the number of subtraction operations required after reordering. This is why sorting is a valid and optimal strategy.
