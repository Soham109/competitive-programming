# Inversion Graph

> [Problem on Codeforces](https://codeforces.com/problemset/problem/1638/C)

## Idea

The key observation is that a connected component in the graph corresponds to a contiguous segment of the permutation $p$ where all elements in the segment form a single "inversion cluster." Specifically, a connected component ends at index $i$ if and only if the maximum value of $p$ in the segment $[1, i]$ equals $i$. This is because all indices up to $i$ are reachable from each other via the inversion edges, and the condition $p_i > p_j$ ensures that the maximum value of $p$ up to $i$ must match the number of elements in the segment.

## Approach

1. Track the maximum value of $p$ encountered so far while iterating through the permutation. Let this value be `mx`.
2. At each index $i$, check if `mx == i + 1`. If true, it means that the elements from the start of the permutation up to index $i$ form a single connected component, and a new component begins after $i$.
3. Increment the component count whenever a new component is identified.
4. Continue this process until the end of the permutation, and the final component count will be the answer.

This approach works because the condition `mx == i + 1` guarantees that all indices in the current segment are connected, and no further connections to indices beyond $i$ are possible.

## Complexity

- **Time:** $O(n)$ per test case, as we iterate through the permutation once to compute `mx` and count components.
- **Space:** $O(1)$, as we only use a few variables to track the maximum and the component count.

## Notes

- The key insight avoids explicitly constructing the graph, which would be computationally expensive. Instead, we leverage the properties of the permutation and the inversion condition to directly count components in linear time.
