# Uninteresting Number

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2050/C)

## Idea

A number is divisible by $9$ exactly when its digit sum is divisible by $9$. The
only allowed move — replacing a digit by its square while it stays a single
digit — is non-trivial for just two digits: $2 \to 4$ and $3 \to 9$. Everything
else either overflows to two digits ($4,\dots,9$) or is fixed ($0,1$). So the
problem reduces to: starting from the base digit sum $S$, can we reach a multiple
of $9$ using increments of $+2$ (per converted $2$) and $+6$ (per converted $3$)?

## Approach

1. Read $n$ as a string and compute $S$, the sum of all digits, counting the
   number of $2$'s as `n1` and the number of $3$'s as `n2` separately.
2. Converting a $2 \to 4$ adds $+2$ to the digit sum; converting a $3 \to 9$ adds
   $+6$. With $a$ converted twos and $b$ converted threes the new sum is
   $S + 2a + 6b$.
3. We need $S + 2a + 6b \equiv 0 \pmod 9$ for some $0 \le a \le n1$,
   $0 \le b \le n2$. Try every pair $(a, b)$ and report `YES` if any satisfies it.

## Complexity

- **Time:** $O(n + n1 \cdot n2)$ per test — one pass to read digits, then a scan
  over conversion counts. Since residues repeat mod $9$, $a, b \le 9$ already
  suffice, making the search effectively $O(n)$.
- **Space:** $O(1)$ — only the two digit counts and the running sum.

## Notes

- Only the digit sum mod $9$ matters, so the positions of the digits are
  irrelevant — just how many $2$'s and $3$'s exist.
