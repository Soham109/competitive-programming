# 2050C

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2050/C)

## Idea

Count the occurrences of digits `2` and `3` in the input number, and compute the sum of the other digits. The key insight is to determine if it's possible to form a valid combination of these counts such that the overall sum is congruent to a specific value modulo $9$.

## Approach

1. Read the input number as a string to access each digit.
2. Initialize a map `mp` to count digits `2` and `3`, and a variable `sum` to accumulate the sum of other digits.
3. For each digit in the string, update `mp` and `sum` accordingly.
4. Call the `possible` function with the counts of digits `2` and `3`, and the computed `sum`.
5. Output `YES` if a valid combination exists, otherwise output `NO`.

## Complexity

- **Time:** $O(n^2)$ due to the nested loops in the `possible` function iterating over all pairs of counts.
- **Space:** $O(1)$ for the fixed-size map to store counts of `2` and `3`.

## Notes

- The calculation of $(2 \cdot k_1 + 4 \cdot (n_1-k_1) + 3 \cdot k_2 + 9 \cdot (n_2-k_2)) \pmod{9}$ is essential for checking the condition efficiently.
