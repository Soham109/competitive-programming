# Crimson Triples

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2238/B)

## Idea

The key observation is that for a triple $(a, b, c)$ to satisfy the given condition, the value of $b$ determines the count of valid triples. Specifically, for a fixed $b$, the number of valid $(a, c)$ pairs is equal to the square of the number of divisors of $b$. This is because $a$ and $c$ must both divide $b$ to satisfy the condition, and the divisors of $b$ are the only candidates for $a$ and $c$. This insight reduces the problem to efficiently computing the sum of squares of divisor counts for all integers from $1$ to $n$.

## Approach

1. **Reformulate the problem using divisors**: For each $b$ in $[1, n]$, the number of valid $(a, c)$ pairs is the square of the number of divisors of $b$. Denote the number of divisors of $b$ as $div(b)$. The total number of crimson triples is then $\sum_{b=1}^n div(b)^2$.

2. **Efficient computation of divisor counts**: Use a sieve-like approach to compute $div(b)$ for all $b \in [1, n]$. For each $i \in [1, n]$, increment the divisor count of all multiples of $i$ by $1$. This works because $i$ is a divisor of all its multiples.

3. **Compute the sum of squares**: Iterate over all $b \in [1, n]$, compute $div(b)^2$ for each $b$, and accumulate the result.

4. **Handle multiple test cases**: Since the sum of $n$ across all test cases is bounded by $2 \cdot 10^5$, precompute the divisor counts and their squares up to the maximum $n$ in all test cases. For each test case, simply sum the precomputed squares up to the given $n$.

## Complexity

- **Time:** $O(n \log n)$ for precomputing divisor counts using a sieve-like approach, where $n$ is the maximum value across all test cases.
- **Space:** $O(n)$ for storing the divisor counts and their squares.

## Notes

- When calculating $div(b)^2$, ensure the result does not overflow. Use `long long` for all arithmetic operations to handle large sums safely.
