# An Alternative Way

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2241/D)

## Idea

The key insight is that the operation alternates between adding and subtracting 1 for consecutive indices, which means it can be used to adjust the difference between the arrays $a$ and $b$ in a cumulative manner. Specifically, the operation preserves the parity of the cumulative difference $pref = \sum (b_i - a_i)$ up to any index $i$. If this cumulative difference ever becomes negative at any point, it is impossible to make $a$ equal to $b$ because the operation cannot "undo" a negative deficit. Thus, the problem reduces to checking whether $pref \geq 0$ at all indices.

## Approach

1. Define a variable `pref` to track the cumulative difference $pref = \sum (b_i - a_i)$ as we iterate through the arrays.
2. For each index $i$, update `pref` by adding $b_i - a_i$ to its current value.
3. If at any point `pref` becomes negative, output "NO" for that test case because it is impossible to make $a$ equal to $b$.
4. If the loop completes without `pref` ever being negative, output "YES" for that test case because the operations can be used to adjust $a$ to match $b$.

## Complexity

- **Time:** $O(n)$ per test case, where $n$ is the length of the arrays. Across all test cases, the complexity is $O(\sum n)$.
- **Space:** $O(1)$, as only a constant amount of extra space is used.

## Notes

- The operation's alternating pattern ensures that the parity of the cumulative difference `pref` is preserved. This is why checking for non-negative `pref` at every step suffices to determine feasibility.
- The solution avoids directly simulating the operation, which would be computationally expensive, by leveraging the mathematical properties of the operation.
