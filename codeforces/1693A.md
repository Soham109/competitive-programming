# Directional Increase

> [Problem on Codeforces](https://codeforces.com/problemset/problem/1693/A)

## Idea

The key insight is that the prefix sum of the array $a$ must satisfy two conditions: (1) it should never drop below zero, and (2) once it becomes zero, it must remain zero for the rest of the array. This is because the operations only allow transferring "units" forward or backward, and the pointer must end at the first element, which implies that any excess units must be balanced out by the end of the traversal.

## Approach

1. Compute the prefix sum of the array $a$ as you iterate through it. Let `pref` denote the running prefix sum.
2. As you compute `pref`, check if it ever becomes negative. If $pref < 0$ at any point, output "No" for the current test case because it is impossible to achieve negative values with the allowed operations.
3. Track whether `pref` has reached zero at any position. If `pref = 0` at some index, set a flag `stopped = true` to indicate that the pointer must now remain at zero for the rest of the array.
4. If `stopped` is true and $pref \neq 0$ at any subsequent index, output "No" because this violates the requirement that the pointer must remain at zero after reaching it.
5. After processing the entire array, check if `pref = 0`. If not, output "No" because the pointer cannot return to the first element with a non-zero prefix sum. Otherwise, output "Yes".

## Complexity

- **Time:** $O(n)$ per test case, where $n$ is the size of the array. Summing over all test cases, the total complexity is $O(T \cdot n)$, where $\sum n \leq 2 \cdot 10^5$.
- **Space:** $O(1)$, as the algorithm uses only a few variables for tracking state.

## Notes

- The key to solving the problem is recognizing the invariant conditions on the prefix sum (`pref`) and ensuring they are maintained throughout the traversal.
- The constraints on the sum of $n$ across all test cases ensure that the solution is efficient enough to run within the time limits.
