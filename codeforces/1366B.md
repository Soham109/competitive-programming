# Shuffle

> [Problem on Codeforces](https://codeforces.com/problemset/problem/1366/B)

## Idea

The key observation is that the position of the $1$ in the array can only spread to indices that are part of overlapping intervals in the given operations. If two intervals overlap, they effectively merge into a single larger interval. By repeatedly merging overlapping intervals that include the initial position of the $1$, we can determine the final contiguous range of indices where the $1$ can end up.

## Approach

1. Start by initializing the range $[lo, hi]$ to $[x, x]$, where $x$ is the initial position of the $1$ in the array.
2. For each operation $(l_i, r_i)$, check if the interval $[l_i, r_i]$ overlaps with the current range $[lo, hi]$. Two intervals overlap if and only if $l_i \leq hi$ and $lo \leq r_i$.
3. If the intervals overlap, merge them by updating $lo = \min(lo, l_i)$ and $hi = \max(hi, r_i)$. This expands the range $[lo, hi]$ to include all indices that can potentially contain the $1$ after the operation.
4. After processing all $m$ operations, the size of the final range $[lo, hi]$ is $hi - lo + 1$, which represents the number of indices $k$ such that $a_k = 1$ is possible.
5. Output the size of this range for each test case.

## Complexity

- **Time:** $O(m)$ per test case, as each operation is processed once. Across all test cases, the total complexity is $O(t \cdot m)$.
- **Space:** $O(1)$, as only a few variables are used to track the range.

## Notes

- The problem is efficient despite $n$ being as large as $10^9$ because the operations only depend on the $m$ intervals, not the size of the array.
- The condition for interval overlap, $l_i \leq hi$ and $lo \leq r_i$, ensures correctness without explicitly simulating swaps.
