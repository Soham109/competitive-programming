# AI Finds Nothing Here

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2240/B)

## Idea

The key insight is that for the matrix to be "clean," every $r \times c$ submatrix must have an XOR sum of $0$. This imposes strict constraints on the values in $a$: adjacent rows and columns must be identical in certain patterns. Specifically, for any $r \times c$ submatrix, all rows within the submatrix must be identical, and all columns within the submatrix must also be identical. This leads to a combinatorial structure where the number of valid matrices depends on the number of independent rows and columns.

## Approach

1. **Identify independent rows and columns:**  
   To satisfy the XOR condition, each group of $r$ consecutive rows must be identical, and similarly for $c$ consecutive columns. This means the first $r-1$ rows and $c-1$ columns are determined by the remaining rows and columns. Therefore, the number of independent rows is $n - r + 1$, and the number of independent columns is $m - c + 1$.

2. **Count valid configurations:**  
   Each independent row or column can independently be either all $0$s or all $1$s, as the XOR condition is satisfied in both cases. Thus, the total number of valid matrices is $2^{(n - r + 1) \cdot (m - c + 1)}$, where each independent row-column pair contributes a degree of freedom.

3. **Efficient modular exponentiation:**  
   Since $n$ and $m$ can be as large as $10^9$, directly calculating $2^{(n - r + 1) \cdot (m - c + 1)}$ is infeasible. Instead, use modular exponentiation to compute the result modulo $998244353$ efficiently.

4. **Output the result for each test case:**  
   For each test case, compute the number of valid matrices using the above formula and modular exponentiation, and output the result.

## Complexity

- **Time:** $O(\log((n - r + 1) \cdot (m - c + 1)))$ per test case due to modular exponentiation. With $t$ test cases, the total complexity is $O(t \cdot \log((n - r + 1) \cdot (m - c + 1)))$.
- **Space:** $O(1)$, as the algorithm uses only a constant amount of extra space.

## Notes

- Modular arithmetic is critical to avoid overflow, as the exponent $(n - r + 1) \cdot (m - c + 1)$ can grow extremely large. The `pow` function in the solution efficiently computes modular exponentiation using repeated squaring.
