# Subset Multiplication

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2124/C)

## Idea

The key observation is that the array $b$ can be derived from a "beautiful array" $a$ by multiplying some subset of its elements by a single integer $x$. Since $a$ is a "beautiful array" (i.e., $a_i \mid a_{i+1}$ for all $1 \leq i \leq n-1$), the ratios $\frac{b_{i+1}}{b_i}$ for consecutive elements in $b$ must also respect this divisibility property. This means that $x$ must account for any violations of divisibility in $b$ while preserving the structure of $a$. The least common multiple (LCM) of the required adjustments to restore divisibility determines a valid $x$.

## Approach

1. Start by iterating through consecutive pairs of elements in $b$. For each pair $b_i, b_{i+1}$, check if $b_{i+1} \bmod b_i = 0$. If this condition holds, the pair is already consistent with the "beautiful" property, so no adjustment is needed.
   
2. If $b_{i+1} \bmod b_i \neq 0$, calculate the factor $f = \frac{b_i}{gcd(b_i, b_{i+1})}$, where `gcd` is the greatest common divisor. This factor represents the minimum multiplier required to make $b_i \cdot f$ divide $b_{i+1}$.

3. Maintain a running least common multiple (`x`) of all such factors $f$ encountered. This ensures that $x$ is a common multiplier sufficient to restore divisibility for all violations.

4. After processing all pairs, output the computed `x`. This guarantees that $b$ could be derived from some "beautiful array" $a$ using this $x$.

## Complexity

- **Time:** $O(n \cdot \log(\max(b)))$ per test case, dominated by the computation of `gcd` and `lcm` for each pair of consecutive elements. Across all test cases, the total complexity is $O(T \cdot n \cdot \log(\max(b)))$, where $T$ is the number of test cases.
- **Space:** $O(1)$, as we only store a few variables for calculations.

## Notes

- The use of `gcd` and `lcm` ensures that the computation of $x$ is efficient and avoids overflow.
- The problem guarantees the existence of a valid $x$, so no additional checks are required for edge cases like $x > 10^9$.
