# Portal

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2200/D)

## Idea

The key insight is that the lexicographically smallest permutation can be obtained by rearranging the elements between the two portals ($p[x+1]$ to $p[y]$, inclusive) such that the smallest element among them is placed at the beginning of this subarray. This is because the lexicographical order is determined by the earliest differing element, so starting with the smallest element ensures optimality. The elements outside the portals remain in their original relative order, but the rearranged subarray can be shifted to the position where it fits lexicographically.

## Approach

1. **Partition the array:** Divide the array into three parts: 
   - The elements before the first portal (`outside` from $p[0]$ to $p[x-1]$).
   - The elements between the two portals (`mid` from $p[x]$ to $p[y-1]$).
   - The elements after the second portal (`outside` from $p[y]$ to $p[n-1]$).

2. **Rearrange the middle segment:** Find the smallest element in `mid` and rotate the subarray such that this smallest element becomes the first element of the subarray. This ensures the lexicographically smallest order for the `mid` segment.

3. **Merge the segments:** Identify the position in the `outside` segment where the rearranged `mid` segment can be inserted to maintain lexicographical order. This position is the first index in `outside` where an element is greater than the first element of `mid`. Insert the rearranged `mid` at this position.

4. **Output the result:** Concatenate the `outside` elements before the insertion point, the rearranged `mid` segment, and the remaining `outside` elements after the insertion point to produce the final permutation.

## Complexity

- **Time:** $O(n)$ per test case, as each partitioning, finding the minimum, and merging operation is linear in the size of the array. Across all test cases, the total complexity is $O(\sum n)$, where $\sum n \leq 2 \cdot 10^5$.
- **Space:** $O(n)$ per test case for temporary storage of the `outside` and `mid` segments.

## Notes

- The key challenge is determining the correct position to insert the rearranged `mid` segment into the `outside` segment. This is done by scanning the `outside` segment to find the first element larger than the smallest element of `mid`.
- The solution avoids unnecessary operations by directly modifying the `mid` segment and merging it with the `outside` segment in a single pass.
