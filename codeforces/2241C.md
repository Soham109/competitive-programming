# RemovevomeR

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2241/C)

## Idea

The key insight is that palindromic substrings of length $\geq 2$ can only reduce the string's length if they contain at least one pair of consecutive distinct characters. This is because deleting one character from a palindromic substring that consists entirely of identical characters (e.g., `000` or `111`) does not break the palindrome property, and such substrings can continue to shrink indefinitely. However, if the string contains at least one transition (e.g., `01` or `10`), the operations can eventually reduce the string to a minimal length of $1$.

## Approach

1. **Count transitions:** Traverse the string and count the number of transitions between `0` and `1` (or vice versa). Let this count be `c`.
2. **Determine the result:**
   - If `c = 0` (the string consists of only `0`s or only `1`s), the minimum achievable length is $1$ because we can repeatedly delete characters from the palindromic substring until a single character remains.
   - If $c \geq 1$, the string contains at least one pair of consecutive distinct characters. In this case, the operations can reduce the string to a minimum length of $1$ if there are enough operations. However, since the problem allows an arbitrary number of operations, the result depends on whether $c = 1$ (minimum length $2$) or $c > 1$ (minimum length $1$).
3. **Output the result:** For each test case, output $1$ if $c > 1$ and $2$ if $c = 1$.

## Complexity

- **Time:** $O(t \cdot n)$, where $t$ is the number of test cases and $n$ is the length of the string. This is because we perform a single traversal of each string to count transitions.
- **Space:** $O(1)$, as we only use a constant amount of extra space.

## Notes

- The key observation about transitions ensures that the solution is both optimal and efficient.
- The implementation avoids unnecessary operations by directly computing the result based on the transition count, making it straightforward and fast.
