# Storming Arasaka

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2238/D)

## Idea

The key insight is that the minimum number of layers $k$ corresponds to the sum of the exponents in the prime factorization of $n$. This works because each prime factor $p$ of $n$ contributes its powers $p^1, p^2, \dots, p^e$ to the divisors of $n$, and these powers must each occupy separate layers due to the problem's constraints. The total number of layers is therefore the sum of the exponents across all prime factors of $n$, plus the number of distinct prime factors minus one (to account for the layering rule).

## Approach

1. **Precompute smallest prime factors (SPF):** Use a sieve-like algorithm to compute the smallest prime factor for every integer up to $10^6$. This allows efficient prime factorization of any number $n$.

2. **Factorize $n$ efficiently:** For each test case, repeatedly divide $n$ by its smallest prime factor (retrieved from the `spf` array). Track the total number of prime divisors (including multiplicities) and the number of distinct prime factors.

3. **Compute the minimum layers:** The total number of layers is the sum of the exponents of all prime factors (total prime divisors) plus the number of distinct prime factors minus one. This formula arises because each distinct prime factor contributes one additional layer due to the layering constraint.

4. **Output the result:** For each test case, print the computed number of layers.

## Complexity

- **Time:** $O(n \log \log n + T \cdot \log n)$, where $n = 10^6$ is the maximum possible value of $n$ and $T$ is the number of test cases. The precomputation of the smallest prime factors uses a sieve-like algorithm with complexity $O(n \log \log n)$. Each test case requires $O(\log n)$ operations to factorize $n$ using the precomputed `spf` array.
- **Space:** $O(n)$ for the `spf` array.

## Notes

- The use of the smallest prime factor (SPF) array ensures that the factorization of $n$ is efficient, avoiding the need for trial division up to $\sqrt{n}$ for each test case.
- The formula for the number of layers is derived directly from the problem constraints: the sum of the exponents ensures all powers of a prime are in separate layers, and the addition of the number of distinct primes minus one accounts for the layering rule.
