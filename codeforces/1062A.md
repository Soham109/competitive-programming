# A Prank

> [Problem on Codeforces](https://codeforces.com/problemset/problem/1062/A)

## Idea

The key observation is that JATC can erase consecutive elements only if the erased segment can be uniquely reconstructed based on the remaining elements and the constraints of the problem. Specifically, the erased segment must consist of consecutive integers, and the endpoints of the erased segment must either be at the boundaries of the valid range $[1, 1000]$ or directly connected to the remaining elements. This ensures that the erased values can be inferred without ambiguity.

## Approach

1. Iterate through the array while maintaining a "current segment" of consecutive integers. Track the start and end of this segment (`rstart` and `rend`) and its length (`cur`).

2. For each new element in the array, check if it extends the current segment (i.e., $a[i] - a[i-1] = 1$). If it does, update `rend` and increment `cur`. Otherwise, evaluate the maximum number of elements that can be erased from the completed segment:
   - If the segment starts at $1$, it is permissible to erase all but one element at the end of the segment.
   - If the segment ends at $1000$, it is permissible to erase all but one element at the start of the segment.
   - Otherwise, it is only permissible to erase all but the first and last elements of the segment.
   Update the maximum erasable count (`ans`) accordingly.

3. Start a new segment with the current element as both `rstart` and `rend`, and reset `cur` to $1$.

4. After the loop, perform the same evaluation for the last segment to ensure no potential erasures are missed.

5. Output the maximum number of erasable elements, stored in `ans`.

## Complexity

- **Time:** $O(n)$ — a single pass through the array to process segments.
- **Space:** $O(1)$ — only a constant number of variables are used.

## Notes

- The edge cases where the segment starts at $1$ or ends at $1000$ are handled explicitly to ensure correctness, as these boundaries allow for additional erasures.
