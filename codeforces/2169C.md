# Range Operation

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2169/C)

## Idea

The key observation is that performing the operation on a segment $\left[l, r\right]$ replaces all elements in that range with the value $l + r$. This means the contribution of the range to the total sum becomes $(r - l + 1) \cdot (l + r)$. To maximize the total sum, we need to compare the gain from performing the operation on any range $\left[l, r\right]$ against the sum of the original array, and choose the best option. The challenge is to efficiently compute the maximum possible gain for all ranges $\left[l, r\right]$.

## Approach

1. Let `sum` track the total sum of the array as we iterate through it. For any range $\left[l, r\right]$, the gain from replacing the range with $l + r$ can be expressed as:
   $$
   \text{gain} = (r - l + 1) \cdot (l + r) - \text{original sum of the range}.
   $$
   Simplifying, this reduces to:
   $$
   \text{gain} = r^2 + r - l^2 - l - \text{sum of elements in } \left[l, r\right].
   $$

2. Rewrite the expression to isolate terms dependent on $r$:
   $$
   \text{gain} = r^2 + r - \text{sum of elements in } \left[l, r\right] + \text{(terms dependent on } l\text{)}.
   $$
   To maximize this, we track the maximum value of $-(l^2 - l + sum of elements before  l)$ as we iterate through the array. This allows us to efficiently compute the best possible gain for any $r$.

3. Iterate through the array, maintaining:
   - `sum`: the cumulative sum of the array up to the current index.
   - `mxvl`: the maximum value of $-(l^2 - l + cumulative sum before  l)$ seen so far.

   For each $r$, compute the potential gain using the formula:
   $$
   \text{current gain} = r^2 + r - \text{cumulative sum up to } r + \text{mxvl}.
   $$
   Update the maximum possible gain (`ans`) accordingly.

4. The final result for each test case is the maximum gain (`ans`) added to the original array sum (`sum`), as performing no operation is also a valid choice.

## Complexity

- **Time:** $O(n)$ per test case, as we perform a single pass through the array while maintaining cumulative values.
- **Space:** $O(1)$, as we only use a few variables for tracking sums and maximums.

## Notes

- The formula for `mxvl` ensures that we efficiently track the best possible contribution from the left boundary $l$ without explicitly iterating over all possible ranges.
- The use of `long long` prevents overflow when computing terms like $r^2$.
