# Creating Keys for StORages Has Become My Main Skill

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2072/C)

## Idea

The key insight is that the bitwise "OR" operation is monotonic: adding more numbers to the array cannot decrease the result. To maximize the `MEX` of the array while ensuring the bitwise "OR" equals $x$, we should include as many distinct integers as possible, starting from $0$ and incrementing upwards, while ensuring that the cumulative bitwise "OR" remains $x$. If fewer than $n$ integers satisfy this condition, we can fill the remaining positions with $0$ or $x$ without affecting the `MEX` or the bitwise "OR".

## Approach

1. Start with an empty array `ans` and a variable `cur\_or = 0` to track the cumulative bitwise "OR" of the array.
2. Iterate over integers $i$ starting from $0$. For each $i$, check if adding it to `ans` would preserve the condition $(cur\_or \lor i) = x$. If true, append $i$ to `ans` and update `cur\_or` to include $i$.
3. Stop the iteration if adding more integers would violate the condition or if $|ans| = n$.
4. If the array has fewer than $n$ elements, append $x$ to `ans` (if necessary) to ensure the bitwise "OR" equals $x$. Then, fill the remaining positions with $0$.
5. Output the array `ans`, which now satisfies both the `MEX` maximization and the bitwise "OR" condition.

## Complexity

- **Time:** $O(n)$ per test case, as we iterate through integers and add them to the array until the conditions are met. Across all test cases, the total complexity is $O(\sum n)$, which is bounded by $2 \cdot 10^5$.
- **Space:** $O(n)$ per test case to store the array.

## Notes

- The condition $(i \lor x) = x$ ensures that $i$ does not introduce any new bits that are not already set in $x$. This is crucial for maintaining the required bitwise "OR".
- If the array is not fully filled after iterating through the valid integers, appending $x$ ensures the bitwise "OR" condition is satisfied, while appending $0$s does not affect the `MEX`.
