# Remove the Ends

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2064/C)

## Idea

The key insight is that the maximum coins collected can be computed by considering the cumulative sums of positive and negative values in the array. Specifically, the optimal strategy is to either take all positive values or split the array at some index $i$, collecting all positive values up to $i$ and all negative values (converted to positive) after $i$. This works because the operation of removing a prefix or suffix ensures that once a segment is removed, it cannot contribute further to the total, so the problem reduces to finding the best split point.

## Approach

1. Compute two prefix sums: `possum[i]` (the sum of all positive values up to index $i$) and `negsum[i]` (the sum of absolute values of all negative numbers up to index $i$).
2. Compute the total sum of all positive values (`psum`) and the total sum of all absolute values of negative numbers (`nsum`).
3. Initialize the answer as the maximum of `psum` (taking all positive numbers) and `nsum` (taking all negative numbers).
4. Iterate over all possible split points $i$ in the array. For each $i$, calculate the coins collected by taking all positive values up to $i$ and all negative values (converted to positive) after $i$. This can be computed as `possum[i] + nsum - negsum[i]`.
5. Update the maximum answer with the value computed in the previous step.
6. Output the maximum coins for each test case.

## Complexity

- **Time:** $O(n)$ per test case, as we compute prefix sums in $O(n)$ and iterate over the array once. Across all test cases, the total complexity is $O(sum of all  n)$.
- **Space:** $O(n)$ per test case for the prefix sum arrays.

## Notes

- The solution avoids directly simulating the removal process by leveraging prefix sums, which efficiently track the cumulative contributions of positive and negative values.
- The use of `long long` for the prefix sums ensures that the solution handles large values of $a_i$ without overflow.
