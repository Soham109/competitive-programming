# 2202B

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2202/B)

## Idea

The goal is to ensure that the characters 'a' and 'b' do not exceed their allowed frequencies in the final string configuration. By maintaining separate logic for even and odd lengths, we can ensure the constraints are satisfied while still meeting the requirements for pairwise distinctness.

## Approach

1. Read the integer $n$ and the string $x$ of length $n$.
2. Check if the count of 'a' exceeds $(n + 1)/2$ or if the count of 'b' exceeds $n/2$. If true, output "NO".
3. If $n$ is even, invoke the `even(n, x)` function; otherwise, invoke the `odd(n, x)` function to validate the string.
4. Output "YES" if the string meets the conditions, otherwise output "NO".

## Complexity

- **Time:** $O(n)$ for counting characters and validating conditions — dominated by string traversal.
- **Space:** $O(1)$ for a constant amount of additional space used.

## Notes

- The algorithm ensures that both even and odd indexed checks correctly validate the string configurations without violating the constraints.
