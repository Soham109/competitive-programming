# 2202B

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2202/B)

## Idea

The key observation is that the string $S$ generated from $T$ must alternate between 'a' and 'b', starting with 'a' if $n$ is odd and starting with 'b' if $n$ is even. This alternation arises because $T$ itself alternates between 'a' and 'b', and removing letters from either end preserves the alternation. Thus, to check if $X$ can be transformed into $S$, we only need to verify that $X$ matches this alternation pattern while respecting the fixed characters and replacing '?' with valid choices.

## Approach

1. **Count constraints:** First, count the occurrences of 'a' and 'b' in $X$. If the number of 'a's exceeds $\lceil n/2 \rceil$ or the number of 'b's exceeds $\lfloor n/2 \rfloor$, output "NO" because $T$ cannot contain enough letters to generate $S$ with those counts.
2. **Determine expected alternation:** Based on the parity of $n$, determine the expected alternation pattern for $S$. If $n$ is odd, $S$ must start with 'a' and alternate as "abab...". If $n$ is even, $S$ must start with 'b' and alternate as "baba...".
3. **Validate $X$ against the pattern:** Iterate through $X$ and compare each character with the expected alternation. For fixed characters ('a' or 'b'), ensure they match the expected value. For '?', treat it as a wildcard that can match either 'a' or 'b'. If any mismatch occurs, output "NO".
4. **Output the result:** If $X$ satisfies the alternation pattern and the counts of 'a' and 'b' are valid, output "YES". Otherwise, output "NO".

## Complexity

- **Time:** $O(n)$ per test case, as we perform a single pass through $X$ to count characters and validate the alternation. With $t$ test cases and $\sum n \leq 200,000$, the total complexity is $O(200,000)$.
- **Space:** $O(1)$, as we only use a few counters and variables.

## Notes

- The alternation check is split into two cases based on the parity of $n$, as the starting character of $S$ depends on whether $n$ is odd or even.
- The wildcard '?' simplifies validation, as it can match either 'a' or 'b', reducing the need for backtracking or complex logic.
