# 2202B

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2202/B)

## Idea

The key observation is that the string $S$ generated from $T$ must alternate between 'a' and 'b', starting with 'a' if $n$ is odd and starting with 'b' if $n$ is even. This alternation arises because $T$ itself alternates between 'a' and 'b', and removing letters from either end preserves the alternation. Thus, to check if $X$ can be transformed into $S$, we only need to verify that $X$ matches this alternation pattern while respecting the fixed characters and replacing '?' with valid choices.

## Approach

1. Count the occurrences of 'a' and 'b' in $X$. If the number of 'a's exceeds $\lceil n/2 \rceil$ or the number of 'b's exceeds $\lfloor n/2 \rfloor$, output "NO" because $T$ cannot contain enough letters to generate $S$ with those counts.
2. The expected pattern of $S$ is fixed by the parity of $n$: if $n$ is odd, $S$ must start with 'a' and read "abab..."; if $n$ is even, it must start with 'b' and read "baba...".
3. Iterate through $X$ and compare each character against the expected alternation. A fixed character ('a' or 'b') must equal the expected value, while '?' is a wildcard that matches either, so any mismatch on a fixed character immediately forces "NO".
4. If $X$ respects the alternation and the counts are within bounds, the assignment is realizable, so output "YES"; otherwise "NO".

## Complexity

- **Time:** $O(n)$ per test case, as we perform a single pass through $X$ to count characters and validate the alternation. With $\sum n \leq 2 \cdot 10^5$, the total work is linear.
- **Space:** $O(1)$, as we only use a few counters and variables.

## Notes

- The alternation check splits into two cases on the parity of $n$, since the starting character of $S$ depends on whether $n$ is odd or even.
- The wildcard '?' simplifies validation, as it can match either 'a' or 'b', removing any need for backtracking.
