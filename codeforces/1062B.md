# Math

> [Problem on Codeforces](https://codeforces.com/problemset/problem/1062/B)

## Idea

The key insight is that the minimum value of $n$ is the product of its distinct prime factors. This is because the `sqrt` operation can only reduce $n$ when it is a perfect square, and the `mul$x$ ` operation can be used to adjust the exponents of the prime factors to make them powers of $2$. The minimum number of operations is determined by the largest power of $2$ greater than or equal to the maximum exponent among the prime factors, plus one additional operation if any exponents need to be increased to reach this power.

## Approach

1. **Prime Factorization:** Decompose $n$ into its prime factors using a sieve-based smallest prime factor (`spf`) array. For each prime factor $p_i$, count its exponent $e_i$ in the factorization of $n$.

2. **Compute the Minimum Value of $n$:** Multiply all distinct prime factors $p_i$ together to get the minimum value of $n$. This is the smallest number that can be obtained by reducing $n$ using the allowed operations.

3. **Determine the Maximum Exponent:** Find the maximum exponent $e_{max}$ among all prime factor exponents. The goal is to make all exponents powers of $2$, as this allows the `sqrt` operation to reduce $n$ as much as possible.

4. **Calculate the Target Power of 2:** Compute the smallest power of $2$ that is greater than or equal to $e_{max}$. This is done using the formula $2^{\lceil \log_2(e_{max}) \rceil}$.

5. **Count Operations:** Determine the number of `sqrt` operations required to reduce the target power of $2$ back to $1$ (equal to $\lceil \log_2(e_{max}) \rceil$). Add one more operation if any of the exponents need to be increased to reach this target power.

6. **Output the Results:** Print the product of the distinct prime factors (the minimum $n$) and the total number of operations.

## Complexity

- **Time:** $O(n \log \log n)$ for building the smallest prime factor (`spf`) array using the sieve of Eratosthenes, and $O(\log n)$ for factorizing $n$, where $n \leq 10^6$. Thus, the total complexity is $O(n \log \log n)$.
- **Space:** $O(n)$ for the `spf` array.

## Notes

- The `ceil_log2` function efficiently computes $\lceil \log_2(x) \rceil$ using bitwise operations, which avoids floating-point precision issues.
- The `spf` array allows efficient prime factorization in $O(\log n)$ time per number, which is crucial for handling the upper limit of $n = 10^6$.
