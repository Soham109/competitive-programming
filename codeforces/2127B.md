# Hamiiid, Haaamid... Hamid?

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2127/B)

## Idea

The key insight is that Hamid's escape time is determined by the distance to the nearest walls on both sides of his starting position. Mani's optimal strategy is to place walls to maximize the minimum distance Hamid must traverse to escape. Conversely, Hamid's optimal strategy is to minimize this distance by always moving toward the closer escape route. Thus, the problem reduces to calculating the minimum distance to escape in the worst-case placement of walls by Mani.

## Approach

1. Identify the nearest walls to Hamid's starting position `x` on both the left and right sides. This is done by scanning leftward and rightward from `x` until encountering walls or the grid boundaries. Let $L$ and $R$ represent the positions of the nearest walls to the left and right, respectively. If no wall exists in a direction, set $L$ or $R$ to the grid boundary.

2. Compute the distances to escape in both directions:
   - To escape left, Hamid must move $x - L$ cells, where $L$ is the position of the nearest wall to the left (or $0$ if there is no wall).
   - To escape right, Hamid must move $R - x$ cells, where $R$ is the position of the nearest wall to the right (or $n + 1$ if there is no wall).

3. Mani's optimal wall placement forces Hamid to take the longer of the two escape routes. To counter this, Hamid will always choose the shorter escape route. Thus, the number of days required for Hamid to escape is $\max(\min(x, n - R + 2), \min(L + 1, n - x + 1))$, where:
   - $\min(x, n - R + 2)$ represents Hamid's escape time if Mani places walls to force him left.
   - $\min(L + 1, n - x + 1)$ represents Hamid's escape time if Mani places walls to force him right.

4. The result for each test case is the value of the above expression.

## Complexity

- **Time:** $O(n)$ per test case, as we perform two linear scans to find the nearest walls. Across all test cases, the total complexity is $O(sum of  n  over all test cases) \leq 2 \cdot 10^5$.
- **Space:** $O(1)$, as we use a constant amount of extra memory per test case.

## Notes

- The problem guarantees that Hamid's starting position is always an empty cell, so we do not need to handle invalid inputs.
- The formula for the escape time ensures that edge cases (e.g., starting near the grid boundaries or with no walls in one direction) are handled seamlessly.
