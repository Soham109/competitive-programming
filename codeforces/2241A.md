# Divide and Conquer

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2241/A)

## Idea

The key observation is that the operation $x := \frac{x}{z}$ is only valid if $z$ is a divisor of $x$ and $z < x$. This means the only possible values for $x$ after each operation are its divisors that are smaller than $x$. To make $x$ exactly equal to $y$, $y$ must be a divisor of $x$ (since $z$ must divide $x$). If $y$ is not a divisor of $x$, it is impossible to reach $y$.

## Approach

1. For each test case, check whether $y$ divides $x$. This can be done efficiently using the modulo operation: $x \% y = 0$.
2. If $y$ divides $x$, output "YES" because $y$ can be reached by repeatedly dividing $x$ by valid divisors.
3. If $y$ does not divide $x$, output "NO" because no sequence of valid operations can reduce $x$ to $y$.

This approach works because the divisors of $x$ form the only valid states $x$ can transition to, and $y$ must be one of them for the transformation to be possible.

## Complexity

- **Time:** $O(t)$ — each test case requires a single modulo operation, which is $O(1)$, so the total complexity is linear in the number of test cases.
- **Space:** $O(1)$ — no additional space is required beyond a few variables.

## Notes

None. The problem is straightforward and does not require special implementation tricks.
