# Another Popcount Problem

> [Problem on Codeforces](https://codeforces.com/problemset/problem/2240/A)

## Idea

The key observation is that the number of set bits in a number $x$ is maximized when $x$ is a power of $2$ minus $1$, such as $1, 3, 7, 15, \ldots$ (binary representations like $1, 11, 111, 1111, \ldots$). This is because all bits up to the highest bit are set to $1$. To maximize the total `popcount` of the sequence, we should aim to use as many such numbers as possible, starting from $1$ and doubling the base value iteratively. The challenge is to fit these numbers into the constraints $\sum a_i \leq n$ and $k$ terms.

## Approach

1. Start with the smallest power of $2$ minus $1$, which is $1$ (binary $1$). Use it $k$ times if possible, as this maximizes the `popcount` contribution while minimizing the sum.
2. Gradually move to larger powers of $2$ minus $1$ (e.g., $3, 7, 15, \ldots$) as long as the sum of the sequence remains $\leq n$. For each such value, determine how many times it can be used without exceeding $k$ terms or the remaining sum.
3. If at any point the remaining sum $n$ is too small to allow $k$ terms of the current value, use as many terms as possible with the current value, then distribute the remaining sum across the remaining terms, ensuring the sum constraint is satisfied.
4. Stop when either $k$ terms are used or the sum constraint is fully utilized. The total `popcount` is the sum of the counts of $1$s from all terms in the sequence.

## Complexity

- **Time:** $O(\log n)$ per test case, as the algorithm iterates through the powers of $2$ up to $n$.
- **Space:** $O(1)$, as it uses only a few variables for computation.

## Notes

- The use of powers of $2$ minus $1$ ensures that the `popcount` is maximized for each term.
- The greedy approach of starting with the smallest values ensures that the constraints are respected while maximizing the total `popcount`.
